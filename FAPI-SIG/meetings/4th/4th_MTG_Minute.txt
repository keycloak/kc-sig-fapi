Summary :

 * FAPI-SIG members
   found in https://github.com/keycloak/kc-sig-fapi/blob/master/members.adoc

 * FAPI-SIG repository
   It has been transferred from jsoss-sig/keycloak-fapi to keycloak/kc-sig-fapi.

 * FAPI-RW project
   Remaining PRs have been merged.
   all conformance tests of both FAPI-RW OP w/MTLS & w/Private Key have been passed against the keycloak in the local environment.

 * FAPI-CIBA project
   Its major 1 task have been broken down into 13 separate subtasks.

 * New Client Policy Official Support project
   It has been proposed for Client Policies to be officially supported.
   Need official support for FAPI-RW security profile so that Client Policies' official support are needed at first.

 * keycloak next release
   Not assure but might be expected in Nov.

 * New Admin REST API (clear JSON Document) for Client Policies.
   Not yet determine whether it support or not for the first milestone of Client Policy Official Support.

 * New Admin Console for Client Policies
   Admin Console itself is still in development phase.
   Not yet determine whether it support or not for the first milestone of Client Policy Official Support.
   Keycloak dev team will bring some updated information on the next web meeting.

 * Migration of Client Registration Policy to Client Policy
   There is the option that at this time we skip migration of Client Registration Policy to Client Policy.
